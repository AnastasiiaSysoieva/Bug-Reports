# Bug-Reports

## Project Description

This repository contains examples of bug reports created during manual testing practice.

Each report includes a clear description, steps to reproduce, expected and actual results, and severity.

## Contents

* Structured defect descriptions
* Screenshots for better understanding of issues

## Bug Report Structure

Each bug report includes:

* Bug ID
* Title
* Steps to reproduce
* Expected result
* Actual result
* Severity / Priority
* Environment
* Screenshot 

## Purpose

This repository was created as part of QA practice to demonstrate the ability to identify, document, and report software defects.
